Intercepting all traffic using DD-WRT

Full visiblity and control over network
November 15, 2025 · 1 min · 165 words · Me | Suggest Changes
Table of Contents
    • What is VLAN

I recently got a burst of motivation and decided to format my Netgear R7000 Nighthawl with new firmware. The router had officially reached end of life in the past centuary and I got tired of dealing with an outdated and insecure software.

There only really two open source options, OpenWRT and DD-WRT.

  1. Created new virtual APs via DD-WRT control panel

  2. Via commandline,

route -n            # shows the current routes used by all the interfaces 
ip route show       # kernel routing table

What is VLAN#

  • all devices on the same LAN are in the same broadcast domain
  • we can setup a VLAN to separate the broadcast domains
brctl addbr br_monitor
brctl addif br_monitor wl0.1                            # iot_deadzone   
brctl addif br_monitor wl1.1                            # fuck-iot-5GHz
brctl addif br_monitor wl1.2                            # my_net ; this one wan't added

ifconfig br_iot 10.0.50.1 netmask 255.255.255.0 up      # assigning IP to br_monitor and turning it on
ip link set br_monitor up                               # turn the br_monitor on (DUPLICATE)

tcpdump -i br_monitor -nn                               # Monitor traffic
